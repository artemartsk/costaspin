## 🎯 Результаты Разработки

Вкладка **Documents** в профиле пациента теперь полностью заменяет текстовую заглушку на полноценный интерфейс генерации юридических форм (GDPR/RGPD). 

### Функционал
1. **Новая зависимость (`jspdf`):**
   Добавлена легковесная библиотека для генерации PDF прямо на стороне браузера. Это означает, что **не нужно** разворачивать сложных бэкендов для хранения/генерации ПДФ-ок. Всё происходит за 1 секунду прямо на устройстве администратора клиники.
   
2. **Компонент `DocumentsTab`:**
   Создан новый чистый интерфейс. Он отображает два ключевых договора для пациента (RGPD Data Processing Consent и Ley 41/2002 Medical Storage Consent). 
   - Если согласие подписано, высвечивается зеленый бейдж `SIGNED` и проставленная дата из вашей БД. 
   - Если согласий нет — висит статус `PENDING`.

3. **Master Consent Record (Сводный договор):**
   Реализована логика согласно гайдлайнам **AEPD**. При нажатии на кнопку «Master Consent Record (PDF)» (кнопка появляется только если есть хотя бы одна подпись) генерируется красивый документ А4 на бланке CostaSpine. В него динамически вшиваются все данные пациента, таймстемпы согласий и версия Privacy Policy.

## 🧪 Инструкция по ручному (QA) тестированию

1. Открой проект (`npm run dev`).
2. Зайди в профиль пациента, у которого есть хотя бы одно подписанное согласие (`data_processing_consent = true` или `medical_data_consent = true`).
3. Перейди во вкладку **Documents**.
4. Убедись, что отображаются зеленые статусы напротив подписанных документов и таймстемпы.
5. Нажми кнопку **"Master Consent Record (PDF)"** в правом верхнем углу вкладки.
6. В браузере должен мгновенно скачаться файл (к примеру, `CostaSpine_Consent_John_Doe.pdf`).
7. Открой скачанный PDF — проверь, что галочки, имя и дата генерации подставлены корректно.
