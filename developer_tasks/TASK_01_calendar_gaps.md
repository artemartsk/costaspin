**Status:** TODO

## Контекст задачи
Отличная работа по добавлению базового Drag-and-Drop и кластеризации оверлапингов в календаре! Однако для запуска пилота медицинской AI-системы на филиале Elviria нам не хватает ряда критических бизнес-фич. 

Календарь должен стать полноценным инструментом для отслеживания эффективности работы ИИ и соблюдения расписания клиник.

## Цели (Acceptance Criteria)

### 1. Глобальный фильтр Локаций (Location Strictness)
В текущей реализации `Calendar.tsx` тянет всех врачей и записи без привязки к клинике. Поскольку у нас жесткое правило *"No cross-branch suggestions"*, администратор должен видеть только свою клинику.
- [ ] Добавить в шапку календаря (рядом с фильтрами врачей и кабинетов) `<Select>` с выбором `Location`.
- [ ] Фильтровать массив данных `useAppointments()` и `usePractitioners()` по выбранному `location_id`.
- [ ] По умолчанию автовыбирать локацию "CostaSpine Elviria" (или первую в списке).

### 2. Маркеры источника брони (Booking Source Transparency)
Администраторам нужно визуально отличать ручные записи от записей, созданных роботами. В базе есть поле `booking_source` (`ai_phone`, `whatsapp`, `manual`, `web`).
- [ ] На карточке записи (рендерится в `Calendar.tsx`) необходимо добавить маленькую иконку или бейдж.
- [ ] Если `booking_source == 'ai_phone'`, показывать иконку робота (или VAPI).
- [ ] Если `booking_source == 'whatsapp'`, показывать зеленую иконку WhatsApp (или Twilio).
- [ ] Если `manual / web`, можно оставлять без бейджа.

### 3. Отрисовка нерабочего времени (Roster & Availability)
В коде сетка захардкожена: `const HOURS = Array.from({ length: 10 }, (_, i) => i + 8); // 08:00 - 17:00`.
- [ ] ОЧЕНЬ ВАЖНО: **Никакого хардкода в `mock-data.ts`!** Мозг AI (`booking-engine`) читает реальное расписание из БД (`practitioner_schedules`). Вы обязаны стягивать данные для UI именно из Supabase.
- [ ] Напишите новый хук (например, `usePractitionerSchedules()`) для запроса данных из таблицы `practitioner_schedules`.
- [ ] Календарь должен визуально «затенять» (серый фон/паттерн штриховки) нерабочие дни и часы специалистов, опираясь на стянутые из базы данные.
- [ ] Заблокировать возможность перетаскивания (drag-and-drop) брони в нерабочую/затененную зону врача.
