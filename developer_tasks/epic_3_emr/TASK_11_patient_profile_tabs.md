# 🛠 TASK 11: Patient Profile Tabbed Navigation (UI Refactor)

**Status:** DONE
**Assignee:** Frontend Developer  
**Epic:** Epic 3 - Unified Patient Record (EMR)

## Контекст задачи
В рамках Эпика 3 мы превращаем страницу профиля пациента (`PatientDetail.tsx`) в полноценную электронную медицинскую карту (EMR). Сейчас все данные (сводка, история визитов, ссылки) выведены на один экран, из-за чего интерфейс скоро будет перегружен, когда мы добавим клинические записи, документы и логи ИИ.
Первой задачей является внедрение системы вкладок (Tabs), чтобы подготовить архитектуру UI к новым модулям.

## Цели (Acceptance Criteria)

### 1. Архитектура и Layout
- Оставить без изменений верхний **Хедер** страницы (кнопка "Назад", Аватар, Имя, Телефон, Email).
- Ниже хедера заменить текущую 2-колоночную сетку на навигационный компонент `<Tabs defaultValue="overview">` (используем `shadcn/ui`). Разместите `TabsList` с 4 вкладками.

### 2. Спецификация Вкладок

#### 🟢 Tab 1: "Overview" (Сводка)
- В эту вкладку необходимо **перенести абсолютно весь текущий контент**, который сейчас находится на странице.
- Сюда переезжает левая колонка (блок Profile stats, Forms Link, Notes) и правая колонка (Upcoming & History Appointments). Сетку (Grid) внутри вкладки можно сохранить.
- Функционал выбора (клика) по карточке визита для открытия выезжающей панели `AppointmentDetailPanel` должен продолжать работать без багов.

#### ⚪️ Tab 2: "Clinical Records" (История болезни)
- Пока это заглушка (Placeholder). Вывести пустой контейнер (со стилизацией вроде пунктирной рамки) с текстом: `Clinical SOAP notes will be displayed here`. 
- Контент появится в следующей задаче `TASK_12`.

#### ⚪️ Tab 3: "Documents" (Документы и Анкеты)
- Пока это заглушка. Вывести контейнер с текстом: `GDPR PDF Forms and signed consents will be listed here`.
- В будущем (в `TASK_13`) сюда будет перенесена генерация PDF на основе заполненных анкет.

#### ⚪️ Tab 4: "Communications" (Логи и Таймлайн)
- Пока это заглушка. Вывести контейнер с текстом: `Unified AI calls, transcripts and WhatsApp timeline will be displayed here`.
- Для `TASK_14`.

### 3. Рефакторинг и Чистота кода
- Чтобы файл `PatientDetail.tsx` не разросся до бесконечности в тысячу строк, вынесите логику и вёрстку первой вкладки в отдельный локальный компонент `PatientOverviewTab` (можно прямо в конец файла `PatientDetail.tsx` или в отдельный файл, если удобно).
- Переиспользуйте готовые компоненты `shadcn/ui` и текущий дизайн системы без придумывания новых стилей.
