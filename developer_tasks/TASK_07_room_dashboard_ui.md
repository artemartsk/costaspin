## Контекст задачи
Нам необходимо внедрить страницу детализации индивидуального кабинета (Room). Этот дашборд будет служить главным инструментом администратора для анализа утилизации (загруженности) комнаты и контроля конфигурации оборудования. Основой интерфейса служит утвержденный макет `[Скриншот UI]`.

## Ожидаемый функционал (Acceptance Criteria)

### 1. Маршрутизация и Layout
- Создать страницу `src/pages/RoomDetail.tsx`. Роутинг: `/rooms/:id`.
- Страница должна поддерживать глобальный `location_id` Elviria.
- **Шапка (Header):** Название комнаты, тип (напр. *Chiropractic Room*), локация и бейдж статуса (зеленый `Available` или красный `In Use / Locked`).

### 2. Правая боковая панель (Configuration)
- Вывести данные из сущности Room:
  - **Supported Treatments:** Список скиллов/услуг, разрешенных в этом кабинете (интеграция с таблицей связей `room_services`).
  - **Equipment:** Статичный лист (или динамичный, если есть связи).
  - **Capacity:** Ограничение по пациентам (по умолчанию `1 patient`).
- Вывести блок **Performance**: метрики загрузки (в % от рабочих часов) и сгенерированная выручка (Revenue) за месяц. 
  *(Формула для MVP: Посчитать успешные (`Confirmed/Attended`) записи кабинета в этом месяце x стоимость привязанной услуги).*

### 3. Расписание (Today's Schedule)
- Секция отрисовывает все `appointments` для текущей комнаты (`room_id`) на сегодняшний день.
- Валидация свободных слотов (`idle slots`): Рассчитать разницу между рабочими часами клиники и забронированным временем, выведя предупреждающий баннер ("X idle slots available today").

### 4. Booking History & AI Badges (КРИТИЧНО)
- Добавить вкладку инфотаблицу (Booking History). Даные: Date & Time, Practitioner, Service, Status.
- **Бизнес-требование:** Обязательно добавить новую колонку **Booking Source**. 
- В этой колонке рендерить компонент бейджа, который мы используем в Календаре (например, 🤖 `Bot`, 💬 `WhatsApp`, 🌐 `Web`, 👤 `Manual`). Это жизненно необходимо для отслеживания окупаемости AI-движка по каждому кабинету.
