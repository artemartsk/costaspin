Компонент работает с четырьмя хуками одновременно (`useRoom`, `useRoomAppointments`, `useRoomMaintenanceLogs`, `useRoomSupportedServices`) и на лету вычисляет показатели Performance (Загруженность / Revenue).

## 📋 Тест-кейсы (TDD / MSW)

### TC-01: Базовый рендер и табы
- **Действие:** Замокать `GET /rest/v1/rooms?id=eq.123` так, чтобы вернулся тестовый кабинет (Massage Room 1).
- **Ожидание:** 
  1. Заголовок кабинета и корректный тип иконки (Armchair/Massage) отображаются.
  2. Переключение табов "Booking History" и "Maintenance Log" работает (меняется видимый контент таблиц).

### TC-02: Динамический подсчет Performance (Revenue & Utilization)
- **Действие:** Замокать `useRoomAppointments` так, чтобы вернулись **3** подтвержденных/посещенных записи на сегодня (например, услуги стоимостью 50€, 50€ и 100€).
- **Ожидание:** 
  - Панель Performance показывает `Revenue: €200`.
  - Панель Utilization показывает `((3 слота / 16 дневных слотов) * 100) = ~19%` (ползунок не превышает это значение).
  - Если добавить отмененную запись (cancelled) на 1000€, метрики **не должны меняться** (отмененные и no-show в расчет Revenue и Utilization попадать не должны).

### TC-03: AI Метрики источника (Booking Source Badges)
- **Действие:** Предоставить мок-данные с бронированиями из разных источников (`manual`, `ai_phone`, `whatsapp`, `web`).
- **Ожидание:** В таблице Booking History все записи корректно мапятся в свои иконки: 
  - `ai_phone` -> Bot + Primary CSS текст
  - `whatsapp` -> MessageCircle + Emerald CSS текст
  - Иконки должны использовать правильные классы без ошибок React Keys.

### TC-04: Умный рендер "Idle Slots"
- **Действие:** Передать ровно 4 активные записи на текущий день.
- **Ожидание:** Над табами появляется виджет-предупреждение: "8 idle slots available today — room is 50% booked".
- **Негативный блок:** Если записей 8 (100% забит) или 0 (пусто), предупреждающий amber-виджет рендериться не должен.
